大家好，今天发布的是 Mac Explorer 1.0.15，一款为 macOS 设计的现代文件管理器。 我们没有把目标定成“重新做一个 Finder”，而是重新思考今天的文件工作：本地文件、开发项目、远程服务器、图片资料和压缩归档，是否可以回到同一个连续的工作空间。 接下来我会用几分钟介绍 Mac Explorer 为什么出现，以及它如何让这些高频工作变得更顺手。

现在处理文件时，我们经常需要在多个工具之间来回切换。 找文件依赖搜索工具；确认代码改动要打开终端或 Git 客户端；管理服务器需要 SFTP 工具；压缩、预览和图片整理又各有自己的入口。 真正消耗时间的不是某一次操作，而是不断切换上下文。Mac Explorer 要解决的核心问题，就是让文件工作重新保持连续。
Introducing Mac Explorer

一个窗口，完成全部文件工作。

熟悉的 Mac 操作，加上真正需要的专业能力。

Mac Explorer 的答案很直接：仍然以文件夹为中心，但把真正需要的能力放回文件夹里。 用户不需要学习新的文件概念，仍然可以用熟悉的前进后退、面包屑、多选、拖放和空格预览。 变化在于，搜索更懂内容，代码状态可以直接看见，远程服务器像本地位置一样打开，压缩和批量任务也不再打断当前工作。
The workspace

文件夹，仍然是中心。

真实软件界面 · 网格视图、信息面板、Git 状态与远程位置
这是 Mac Explorer 的实际软件界面。 左侧保留了 macOS 用户熟悉的最近使用、桌面、文稿、下载、图片和音乐，同时加入磁盘、应用程序、废纸篓、远程服务器、AI 智能与收藏夹。 中间仍然是文件列表或网格视图，支持排序、分组和多选操作。文件图标右下角的圆形徽标直接显示 Git 状态。 右侧信息面板会跟随当前选择，展示文本、图片和文档预览，并提供文件信息、标签与快捷操作。整个产品始终围绕“正在浏览的文件夹”展开。

Mac Explorer 的搜索分为两个层次。 画面中展示的是全局搜索：输入少量关键词，就能从不同目录中集中列出匹配文件，并显示完整路径，减少逐层翻找。底层通过 SQLite FTS5 索引文件名、路径和可检索内容；当索引不可用时，还可以回退到文件系统搜索。 第二层是本地图片理解。应用使用 macOS Vision 能力在后台分析图片，识别人脸、场景、物品、动物、地点、日期以及图片中的文字，并把结果组织成人物、分类、地点、日期和文字搜索等虚拟文件夹。 AI 分析可以在设置中关闭，任务会显示在后台任务面板中；图片资料仍留在用户自己的 Mac 上。
Built for real work

开发工作，不必离开文件夹。

Git 状态直接显示在文件图标上
连接、保存并直接浏览远程目录
对于开发项目，Mac Explorer 把两个最常切换的能力带进文件管理器。 第一是 Git 状态。应用会读取当前仓库状态，并用圆形徽标显示已提交、修改、新增或未跟踪等状态。浏览目录时不用先打开终端，就能知道哪些文件正在变化。 第二是 SFTP。画面中可以看到实际的远程服务器连接窗口，以及连接后直接展示的服务器目录。远程服务器会作为侧边栏中的一种位置存在，可以保存多个服务器，使用密码或私钥认证，浏览、上传、下载和编辑远程文件。 连接服务还会暴露断线、重连和重连失败状态，使远程目录更接近稳定的本地工作体验。

实际使用中，很多文件并不需要打开应用才能判断内容。 Mac Explorer 的信息面板可以直接预览文本、图片、PDF 首页、SVG 和系统可生成缩略图的文档。截图里选中 README.md 后，右侧会直接显示 Markdown 文本内容。 面板同时提供文件大小、类型、创建与修改时间、图片 EXIF、系统标签和自定义标签。对于图片，还可以直接执行 OCR 并复制识别出的文字。 压缩、解压、复制、移动和 AI 分析等长任务会进入统一后台任务面板，显示运行、完成和失败状态，并允许取消任务。这些功能共同减少了“打开别的应用确认一下”的次数。

把这些能力放进真实场景，会更容易理解 Mac Explorer 的价值。 开发者可以在项目目录里查看 Git 改动、全文搜索代码、预览文档，并直接整理发布归档。 运维或开发团队可以连接远程服务器，上传构建产物、修改配置文件和下载日志，不必在本地文件管理器与 SFTP 客户端之间切换。 设计师和内容用户则可以依靠缩略图、Quick Look、评分、标签和 AI 分类整理大量截图、照片与资料。

这里不是把 Mac Explorer 和 Finder 做简单的功能输赢比较。 Finder 仍然是优秀的系统文件管理器。Mac Explorer 面向的是需要同时处理开发项目、远程目录、大量资料和复杂批量任务的用户。 它真正替代的不是 Finder，而是“文件管理器加终端、加 Git 客户端、加 SFTP 客户端、加压缩工具、加图片整理工具”的多窗口工作流。 产品价值可以概括为一句话：保留熟悉的文件操作，把散落的专业能力集中起来。
